CLI & Installation

Install locally and classify samples from the command line.

Requirements Installation classify classify-batch check-pole serve Input formats Output

Requirements

  • Python ≥ 3.11
  • uv (recommended) or pip

Installation

From PyPI (when published)

uv pip install ec-molsubtype

From source

git clone https://github.com/viktorlj/ec-molsubtype.git
cd ec-molsubtype
uv venv && source .venv/bin/activate
uv pip install -e .

After installation the ec-molsubtype command is available in your shell.

ec-molsubtype --help

classify

Classify a single sample from a MAF file.

ec-molsubtype classify sample.maf
Argument / OptionDescription
MAF_PATHPath to the MAF file (required)
--metadata, -mJSON metadata sidecar (sample_id, tmb, msi_pct, etc.)
--segCBS segmentation file (.seg) for FGA computation
--output, -oOutput file path. Writes JSON report to file instead of stdout
--msi-thresholdMSI percentage threshold for MSI-H call (default: 20.0)
--human, -HPrint human-readable summary instead of JSON

Examples

# JSON output to stdout
ec-molsubtype classify patient_01.maf

# With metadata sidecar and SEG file
ec-molsubtype classify patient_01.maf -m patient_01.json --seg patient_01.seg

# Human-readable output
ec-molsubtype classify patient_01.maf -H

# Save JSON report to file
ec-molsubtype classify patient_01.maf -o result.json

classify-batch

Classify all .maf files in a directory and write a summary TSV.

ec-molsubtype classify-batch --input-dir ./samples/ --output results.tsv
OptionDescription
--input-dirDirectory containing .maf files (required)
--metadata-tsvTSV with per-sample metadata (sample_id column must match filenames)
--output, -oOutput TSV file (default: results.tsv)
--msi-thresholdMSI percentage threshold (default: 20.0)

Example with metadata

ec-molsubtype classify-batch \
  --input-dir ./maf_files/ \
  --metadata-tsv samples_metadata.tsv \
  --output cohort_results.tsv

check-pole

Quick lookup of a POLE variant against the curated pathogenicity database.

ec-molsubtype check-pole "p.P286R"
Argument / OptionDescription
VARIANT_STRPOLE variant in HGVSp short format, e.g. p.P286R (required)
--tmbOptional TMB value (mut/Mb) to evaluate secondary evidence for VUS

Examples

# Tier 1 hotspot
ec-molsubtype check-pole "p.P286R"

# VUS with TMB for secondary evidence check
ec-molsubtype check-pole "p.D275G" --tmb 45.0

serve

Start the web interface locally. This launches the same application you are viewing now.

ec-molsubtype serve
OptionDescription
--hostBind address (default: 0.0.0.0)
--portBind port (default: $PORT or 8000)
--reloadAuto-reload on code changes (development)

Input formats

MAF file (required)

Tab-separated MAF format. Comment lines starting with # are stripped. Minimum required columns:

Hugo_Symbol Chromosome Start_Position End_Position Reference_Allele Tumor_Seq_Allele2 Variant_Classification HGVSp_Short t_alt_count t_ref_count

Optional: ClinVar_Classification, SIFT, PolyPhen

Metadata sidecar (optional)

JSON file with per-sample information. All fields are optional:

{
  "sample_id": "PATIENT_01",
  "tmb": 285.3,
  "panel_size_mb": 1.5,
  "msi_pct": 0.0,
  "fraction_genome_altered": 0.05,
  "signature_weights": {"SBS10a": 0.45, "SBS10b": 0.22}
}

SEG file (optional)

CBS segmentation output for fraction genome altered (FGA) computation. Standard 6-column format: sample, chromosome, start, end, num_probes, segment_mean.

Output

JSON (default)

Full classification result with primary subtype, confidence, classification path, secondary evidence, clinical notes, and flags. Written to stdout or to file with -o.

Human-readable (-H)

Compact summary for quick terminal review.

Batch TSV

One row per sample with columns: sample_id, primary_subtype, confidence, flags.
